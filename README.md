# 就活データ横比較

有価証券報告書だけを出典に、同業他社を1つの表に並べる静的サイト。

平均年収も男女の賃金の差異も、1社ずつなら調べれば出てくる。しかし**同業7社を1つの表に並べたもの**は
どこにもない。就活情報サイトは企業から広告費を受け取る側なので、企業を不利に並べられないからである。
このサイトは掲載企業から1円も受け取らない。

対象：20業界74社・計367件の有価証券報告書（直近5期）。決算期は3月・12月・2月・8月が混在する。

## 設計上の鉄則

1. **数値をClaudeに読ませない。** XBRL/CSVから機械的に抽出する。Claudeは文章化しか担当しない。
2. **有報PDFが正本、XBRLは照合相手。** XBRLの標準タグ `NumberOfEmployees@NonConsolidatedMember` は
   有報の表に載る従業員数とは限らず（三菱商事・双日・三菱地所・SOMPO HDなどで不一致）、
   **平均年間給与の分母はその表の値**である。`verify_employees.py` が全367件をPDFと突き合わせる。
   PDFの読み方は2通り用意し、テキスト順の読み取りがXBRLのどちらのタグとも一致しないときだけ
   セル座標方式で読み直す（多行表の商船三井・スカイマーク、転置表の本田技研で必要）。
   どちらの読み方でもXBRLと一致しなければ、その数字は**公開しない**。
3. **データが無い項目は「非公表」と書く。** 0や平均値で埋めない。埋めた瞬間に唯一の資産（信頼）を失う。
4. **DBを作らない。** 数MBのJSONをGitに直接コミットする。Git履歴がそのまま数値の監査証跡になる。

## 誠実に見せるための規則（`generate.py` が守る）

- **持株会社の平均年収は本体の数字**であって事業会社（三菱UFJ銀行など）の水準ではない → 警告を出す
- **「男女の賃金の差異」は男性を100としたときの女性の賃金の割合**であって「差」ではない
- **男性育休取得率には2方式ある**（71条の6第2号方式は育児目的休暇を含み高く出る）→ 方式を併記し順位を付けない
- **従業員数は連結と提出会社を分ける**。平均年間給与の分母は提出会社
- **算定基準が変わった年度をまたぐ増減率は出さない**（三井不動産・三菱地所）。有報の注記をそのまま掲示する
- **3年以内離職率は載せない。** 大手企業は公的に開示しておらず、しょくばらぼにも掲載がない（全掲載148,228社のうち埋めているのは1.8%＝中小企業層のみ）
- **最新の有報が他社より一世代古い会社は「古い期」と明示する**（アサヒGHDは2025年12月期を未提出）

## 使い方

```bash
python fetch_edinet.py find --from 2021-06-01 --to 2026-07-09   # 書類一覧 → data/doc_index.csv
python fetch_edinet.py get --types 5 2                          # XBRL(CSV) と PDF を取得（1req/秒）
python verify_employees.py                                      # 有報PDFと突き合わせ → data/employees_verified.csv
python extract.py                                               # 機械抽出 → data/companies/*.json
python generate.py                                              # 静的サイト → site/
python write_prose.py --submit / --collect                      # 任意。Claudeが解説文だけ書く（数値は渡すだけ）
```

`write_prose.py` を実行しなくてもサイトは完全に成立する。文章は装飾であり、商品は数字の並びである。

## 出典とライセンス

- データ：EDINET閲覧（提出）サイト（金融庁）
- ライセンス：[公共データ利用規約（PDL1.0）](https://www.digital.go.jp/resources/open_data/public_data_license_v1.0)
- 本リポジトリの数値は、上記の有価証券報告書を当プロジェクトが機械的に抽出・整形したものです

EDINET利用規約は「短時間における大量のアクセス」を禁じている。`fetch_edinet.py` は全リクエストを
**1件/秒**に強制し、取得済みの書類は再取得しない。アクセスログは `logs/edinet_access.log` に残る。

数値は公的な一次情報から機械的に抽出したものだが、正確性を保証するものではない。
最終的な判断は必ず原典（有価証券報告書）を確認すること。
